# Launcharr Release Announcement

## Headline

Launcharr `v0.2.39` is now available

## Short Summary

Launcharr `v0.2.39` improves Romm `Recently Added` reliability and ordering, adds a configurable Romm probe-limit setting, and refreshes the top-bar controls by moving theme switching into the brand menu and adding a support heart link.

## What's New

- Added Romm `Recently Added Probe Limit` (50-200) in Romm settings to tune API probe size for large libraries.
- Lowered the default Romm recently-added probe size to `50` to reduce timeout risk.
- Updated Romm recently-added ordering to prioritize parsed timestamps for more consistent newest-first results.
- Moved the theme toggle into the Launcharr brand menu (`Theme` action).
- Replaced the top-right theme toggle button with a Buy Me a Coffee heart link.

## Why It Matters

- Romm recently-added retrieval is more tunable for slower/large-library setups without code changes.
- Romm recently-added order is more consistent across Launcharr surfaces when upstream responses are not strictly ordered.
- The top bar is cleaner while still keeping theme switching easy to access.

## Upgrade Notes

- Docker image: `mickygx/launcharr:v0.2.39`
- Supplementary tag: `mickygx/launcharr:development` (if selected during release flow).
- No migration steps required. Hard refresh browser assets after upgrade.

## Documentation

- Changelog: `docs/release/releases/v0.2.39.md`

## Known Issues

- Romm endpoint compatibility/performance still varies by Romm version and library size. If `Recently Added` retrieval is unreliable, reduce the Romm probe limit (minimum `50`) and verify which Romm API endpoints succeed in your environment.
