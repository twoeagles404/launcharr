# Changelog

## [v0.2.39] - 2026-02-22

### Added

- Added a Romm `Recently Added Probe Limit` setting (range `50-200`) in the shared Romm settings UI, available in both `App > Romm > Settings` and `Settings > Apps > Romm`.

### Changed

- Moved the theme toggle action into the top-left Launcharr brand menu (alongside Starfield and Maximize/Minimise controls).
- Replaced the top-right theme toggle button with a Buy Me a Coffee heart link.
- Updated the support heart styling to use a glow treatment and larger tap target.
- Lowered the default Romm recently-added API probe size from `200` to `50` (now configurable per Romm app instance).
- Updated Romm recently-added ordering to sort by parsed timestamp before upstream/source order tie-breaks for more consistent newest-first results.

### Fixed

- Improved Romm `Recently Added` ordering consistency between the Romm app overview and backend API normalization when upstream responses are not already in strict newest-first order.
- Reduced likelihood of Romm recently-added retrieval timeouts on large libraries by using a smaller default probe limit.

### Security

- None.

### Deprecated

- None.

### Removed

- Removed the dedicated top-right light/dark mode button (theme toggle remains available in the brand menu).

### Migration Notes

- No schema/data migration required.
- Existing Romm configs continue to work. The new probe-limit setting is optional and defaults to `50` when left blank.
- Browser hard refresh is recommended after upgrade so updated UI scripts/styles are loaded.

### Verification Checklist

- [ ] Top-right status area shows the Buy Me a Coffee heart link (and no standalone theme toggle button).
- [ ] Brand menu includes a `Theme` action and successfully switches Light/Dark mode.
- [ ] Romm settings page shows `Recently Added Probe Limit` under the admin Romm access section (range `50-200`) in both settings surfaces.

- [ ] Romm `Recently Added` ordering remains newest-first and consistent across Launcharr surfaces for the same data set.
