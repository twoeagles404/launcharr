# Changelog

## [v0.2.44] - 2026-03-02

### Added

- Added a new `Launcharr` system widget (`deployment-summary`) with:
  - online/offline/total app counts
  - app-up percentage status chip
  - status colors (`90-100%` green, `50-89.9%` orange, `<50%` red)
  - settings for metric columns and per-stat visibility.
- Added backend deployment-summary API: `GET /api/widget-deployment-summary`.
- Added built-in `QNAP` app definition and widget support:
  - default app catalog entry (`config/default-apps.json`)
  - app icon (`public/icons/qnap.svg`)
  - stats adapter (`CPU`, `Memory`, `Volume`).
- Added API-key reveal/hide controls in app settings forms, including wider API-key input layout.

### Changed

- Updated settings widget-canvas behavior for `Launcharr` deployment summary cards to open configuration on card click (matching other widget UX; no explicit edit pencil for this widget).
- Updated settings app panel list generation to include all non-removed apps directly, improving app discoverability in settings navigation.
- Updated sidebar/menu normalization so app sidebar sections always keep at least `admin` visibility by default.

### Fixed

- Fixed Immich recently-added module loading by adding compatibility for newer/alternate Immich API response shapes and thumbnail URL variants.
- Fixed qBittorrent widget false-offline scenarios by adding WebUI login flow support and reusing returned session cookie for stats calls.
- Fixed widget status access checks for widget-only apps (for example `QNAP`) by allowing role-visible widget-bar membership as an access path.
- Fixed deployment-summary target counting to include apps with stat-type fallback metadata.
- Fixed custom icon uploads not appearing in `Settings -> Custom -> Images` by correcting icon save/delete paths to the runtime public icon directory.
- Fixed custom icon upload/delete CSRF handling in settings image forms.
- Added icon-upload error diagnostics and migration for previously misplaced custom icon files.

### Security

- Added setup endpoint rate limiting for `POST /setup` (failed-attempt protection).
- Added rate limiting for `GET /api/plex/pin/status` to reduce polling abuse.
- Tightened Plex callback/status PIN handling to use server session PIN only (query pin is no longer accepted).

### Migration Notes

- No schema migration required.
- Rebuild/restart Launcharr and hard refresh browser assets once after upgrade.
- For qBittorrent widgets, set `username` and `password` in app settings if your WebUI requires auth.

### Verification Checklist

- [ ] Launcharr deployment-summary widget renders in both dashboard and settings widget preview.
- [ ] Deployment-summary widget config allows column count and stat visibility changes.
- [ ] Deployment-summary widget shows expected online/offline/total counts and percentage color thresholds.
- [ ] Immich recently-added photos load and thumbnails render.
- [ ] qBittorrent widget reports status correctly when WebUI auth is enabled.
