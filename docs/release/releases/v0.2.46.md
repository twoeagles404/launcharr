# Changelog

## [v0.2.46] - 2026-03-11

### Added

- Added built-in [Curatorr](https://github.com/MickyGX/curatorr) support, a music discovery and playlist curation app, with dedicated icon assets and default sidebar launch sections for Dashboard, History, Artists, Discover, Tracks, Playlists, and admin-only Settings.
- Added role-aware app submenu normalization so Launcharr can expose custom launch destinations directly in the sidebar.

### Changed

- Updated app launch routing so apps that hide the default Launch link redirect `/apps/:id/launch` to the configured default submenu page.
- Updated the launch view to surface the active submenu label in the page title, window bar, and sidebar active state.
- Updated embedded Curatorr launches to pass Launcharr theme and square-corner preferences into the iframe and resync them after appearance changes.

### Fixed

- Fixed invalid or unauthorized app `?page=` launch targets so they fall back to the configured default submenu instead of leaving hidden routes stranded.
- Fixed embedded Curatorr launches drifting out of sync with Launcharr day/night and square-corner preferences after runtime theme changes.

### Migration Notes

- No schema migration required.
- Rebuild/restart Launcharr so the new app metadata and icon assets are included in the image.
- Hard refresh browser assets once after upgrade if the updated Launcharr icon does not appear immediately.

### Verification Checklist

- [ ] Curatorr can be restored from built-in apps and shows the expected sidebar subpages.
- [ ] `/apps/curatorr/launch` redirects to the configured default subpage and each allowed subpage opens correctly.
- [ ] Curatorr `Settings` subpage is visible to `admin` and hidden from lower roles.
- [ ] Launcharr day/night and square-corner changes propagate to the embedded Curatorr iframe after load.
- [ ] Existing app overview/launch navigation remains unchanged for apps without submenu links.
