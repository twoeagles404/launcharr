# Launcharr Release Announcement

## Headline

Launcharr `v0.2.47` is now available

## Short Summary

Launcharr `v0.2.47` fixes Plex multi-instance settings/token handling so secondary Plex apps resolve and save their own server credentials correctly.

## What's New

- Added targeted Plex token-resolution helpers and test coverage for primary and secondary Plex instances.
- Updated Plex settings actions to request token and machine values for the selected Plex app instance.
- Updated Plex dashboard/settings detection so instance IDs such as `plex-2` receive the same Plex-specific behavior as the primary app.

## Why It Matters

- Admins running multiple Plex servers can now retrieve and save credentials per instance without cross-wiring tokens from the primary app.
- Secondary Plex instances now behave consistently across settings, dashboard modules, and launch routing.
- The new tests reduce the risk of future regressions in Plex multi-instance support.

## Upgrade Notes

- Docker image: `mickygx/launcharr:v0.2.47`
- Supplementary tag: `mickygx/launcharr:development`
- No migration steps required.
- Rebuild/restart Launcharr after upgrade and re-check secondary Plex token/machine fields if they were previously captured from the wrong instance.
