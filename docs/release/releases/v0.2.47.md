# Changelog

## [v0.2.47] - 2026-03-12

### Added

- Added Plex settings token-resolution helpers that can resolve the selected Plex instance from `appId` instead of assuming only the primary `plex` app exists.
- Added automated tests for Plex multi-instance app selection and token fallback behavior in `src/test/api-plex.test.js`.

### Changed

- Updated Plex-specific UI and dashboard logic to use base-app detection so Plex instance IDs such as `plex-2` are treated as Plex across launch routing, dashboard modules, and settings layouts.
- Updated `Get Plex Token` and `Get Plex Machine` settings actions to send the current Plex app ID with each request.
- Updated Plex token validation in app settings saves so secondary Plex instances follow the same JWT-guard rules as the primary instance.

### Fixed

- Fixed secondary Plex instances accidentally reusing the primary Plex session server token when resolving settings credentials.
- Fixed Plex dashboard modules and deep-launch/search behavior only activating for the literal `plex` app ID instead of all Plex instances.

### Migration Notes

- No schema migration required.
- Rebuild/restart Launcharr so the updated Plex settings and dashboard logic is loaded.
- Re-open secondary Plex app settings and refresh Plex token/machine values if those fields were previously populated from the wrong instance.

### Verification Checklist

- [ ] `Get Plex Token` on the primary Plex app still resolves a usable server token.
- [ ] `Get Plex Token` and `Get Plex Machine` on a secondary Plex instance (for example `plex-2`) resolve values for that instance, not the primary Plex server.
- [ ] Plex-specific settings layout/styling appears for both the primary Plex app and Plex instances.
- [ ] Plex dashboard modules, search/deep-launch behavior, and related app detection still work for configured Plex instances.
- [ ] `npm test` passes, including `src/test/api-plex.test.js`.
