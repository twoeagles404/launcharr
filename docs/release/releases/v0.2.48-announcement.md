# Launcharr Release Announcement

## Headline

Launcharr `v0.2.48` is now available

## Short Summary

Launcharr `v0.2.48` adds built-in unRAID support with a dedicated widget integration and fixes API-key settings visibility across both app-settings surfaces.

## What's New

- Added built-in `unRAID` app support with its own default metadata and icon.
- Added an unRAID widget that reads CPU, memory, array usage, and alert metrics from the Unraid GraphQL API.
- Fixed `Settings -> Apps` so unRAID shows the same API-key form as the dedicated per-app settings page.

## Why It Matters

- Admins can now add unRAID as a first-class Launcharr app instead of treating it as a generic custom entry.
- Widget-bar deployments can surface key unRAID health indicators alongside existing NAS and infrastructure widgets.
- The settings fix removes a confusing mismatch between the global apps page and the direct app settings page.

## Upgrade Notes

- Docker image: `mickygx/launcharr:v0.2.48`
- Supplementary tag: `mickygx/launcharr:development`
- No migration steps required.
- Rebuild/restart Launcharr after upgrade, then add an Unraid API key before enabling the widget.

## Known Issues

- The unRAID widget depends on the Unraid GraphQL API shape; if a deployment exposes a different schema, the widget may stay offline until that schema is mapped.
