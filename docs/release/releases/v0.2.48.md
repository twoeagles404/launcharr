# Changelog

## [v0.2.48] - 2026-03-16

### Added

- Added built-in `unRAID` app support with default catalog metadata, a dedicated icon asset, and a server-oriented multi-instance placeholder.
- Added an `unraid` widget/stat-card integration for widget bars using the Unraid GraphQL API and `X-API-Key` authentication.
- Added unRAID widget metrics for CPU, memory, array usage, and alert counts.

### Changed

- Updated Launcharr widget stat-type registration so unRAID is available anywhere widget-enabled app families are listed.
- Updated shared app settings copy to document where to obtain an Unraid API key.

### Fixed

- Fixed `Settings -> Apps` so unRAID shows the API-key access card there as well as on the dedicated app settings page.
- Improved unRAID widget compatibility by trying multiple GraphQL memory field shapes (`free`/`total`, `percentage`, `usedPercentage`, `usage`) before failing the probe.

### Migration Notes

- No schema migration required.
- Rebuild/restart Launcharr so the new built-in app metadata, icon asset, and widget integration are loaded.
- Add or update an Unraid API key in Launcharr app settings before enabling the unRAID widget in a widget bar.

### Verification Checklist

- [ ] `unRAID` appears in the built-in app catalog and restores correctly from `Settings -> Apps`.
- [ ] Both `Settings -> Apps` and `/apps/unraid/settings` show the unRAID API-key access form.
- [ ] A configured unRAID widget returns CPU, memory, array, and alert metrics instead of `unknown`.
- [ ] Existing QNAP widget behavior remains unchanged.
- [ ] `node --check src/index.js` and `node --check src/routes/api-specialty.js` pass.
